## 七、核心差异总结

### 7.1 推断风格对比

| 维度 | GPT-5.4 | DeepSeek v4 Pro |
|------|---------|-----------------|
| **推断策略** | 严谨保守，区分证据与推断 | 敢于推断，基于惯例合理猜测 |
| **输出长度** | 极长（~7,000 字） | 较短（~2,000 字） |
| **结论明确性** | 证据不足时不给明确结论 | 即使推断也给出明确结论 |
| **覆盖完整性** | 高，不易遗漏关键点 | 中，易遗漏部分关键点 |
| **Judge 偏好** | 长篇分析有时被视为"未直接回答" | 简洁推断更易被接受 |

### 7.2 能力对比

| 能力 | 胜者 | 说明 |
|------|------|------|
| **推断全面性** | GPT-5.4 | 能覆盖更多参考答案关键点 |
| **推断准确性** | 相当 | 两者都有推断对和推断错的时候 |
| **推断简洁性** | DeepSeek | 输出精炼，不被 judge 视为冗长 |
| **推断胆量** | DeepSeek | 更愿意在证据不足时做推断 |
| **检索利用效率** | GPT-5.4 | 能从相同检索中提取更多信息 |

### 7.3 正确率差异归因

**GPT-5.4 比 DeepSeek 高 4.3%（87.3% vs 83.0%）的原因：**

1. **覆盖更完整**（+19 题）: GPT-5.4 能从相同检索中覆盖更多关键点
2. **推断更谨慎**（-10 题）: GPT-5.4 有时过于谨慎，不给明确结论而失分
3. **净收益**: +9 题 = 4.3% 正确率

## 八、关键结论

1. **检索完全相同**：两个模型的差异纯粹是回答风格和推断策略的差异，与检索无关。

2. **GPT-5.4 是"全面型"**：输出量是 DeepSeek 的 3-5 倍，覆盖更完整，但 Judge 有时认为"未直接回答"。

3. **DeepSeek 是"简洁型"**：输出精炼，敢于推断，但容易遗漏关键点。

4. **Judge 偏好影响结果**：Judge 似乎更偏好简洁直接的回答（即使基于推断），而非长篇分析（即使更全面）。

5. **两者都错的 13 道题是真正的检索盲区**，需要优化检索召回。

6. **性价比**：GPT-5.4 和 DeepSeek 时延相近（~140s），但 GPT-5.4 正确率略高（87.3% vs 83.0%）。若考虑 Token 成本，DeepSeek 的 completion token 更高（3,874 vs 2,140），但 GPT-5.4 的推理速度更慢。

## 九、对系统优化的启示

### 9.1 Prompt 工程

当前 answer_generation prompt 要求"按 4 个步骤分析"，这导致：
- GPT-5.4 严格遵循，输出极长
- DeepSeek 也遵循，但输出相对简洁
- Judge 对长篇分析有偏见

**建议**：简化 prompt，要求"直接给出结论，再简要说明依据"，而非强制 4 步分析。

### 9.2 Judge 评判标准

Judge 对"推断"的接受度不一致：
- DeepSeek 说"未找到但推断是 X" → 判对
- GPT-5.4 说"未找到，分析如下但无结论" → 判错

**建议**：明确 Judge 标准，对"基于合理推断的回答"给予一致评分。

### 9.3 模型选择策略

| 场景 | 推荐 |
|------|------|
| 追求最高正确率 | GPT-5.4（87.3%） |
| 追求简洁输出 | DeepSeek（~2,000 字/题） |
| 成本敏感 | GPT-5.4（completion token 更低） |
| 速度敏感 | 两者相近（~140s），都不理想 |
