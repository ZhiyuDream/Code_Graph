# 用户偏好配置

> 此文件记录实验规范和用户偏好，每次对话前优先读取。

---

## 核心原则（最高优先级）

### 1. 实验正确率优先
- **不考虑时长限制**：测试以确保正确率和完整性为第一目标
- **不简化测试流程**：样本量、对照组、评估指标必须完整
- **后台任务**：长耗时操作用后台运行，而非削减测试范围

### 2. 并行化执行
- QA 推理：并行 20 个问题
- Judge 评估：并行 20 个
- 最大化利用资源，减少等待时间

### 3. 记录与追溯
- 每次改进需记录到 `design/` 目录
- 必须找 good case 和 bad case
- 实验必须和 baseline 对比

## 实验规范

### 评估指标（必须报告）

| 指标 | 说明 |
|-----|------|
| LLM 分数 | 0-1 打分，以及二元正确/错误 |
| Token 消耗 | 总 token 使用量 |
| 延迟 | 总耗时、平均每题耗时 |
| 工具调用轨迹 | 每步调用什么工具，参数和结果 |
| Evidence 召回率 | 相关证据是否被召回 |
| 准确率对比 | vs 上次改进 / vs baseline |

## 历史实验记录

### 2025-04-19: 文件级扩展实验

**改进内容**: 从语义搜索召回的函数所在文件中，扩展召回更多同文件函数

**小样本验证结果** (5题):
| 策略 | 平均相关度 | 提升 |
|-----|-----------|------|
| baseline | 0.126 | - |
| always_expand | 0.170 | +35% |
| adaptive (LLM决策) | 0.164 | +30% |

**关键发现**:
- 召回率提升 5x (10→50 函数)
- 捕获了语义搜索遗漏的关键函数: decode_utf8, is_digit_char, is_word_char, parse_hex
- 噪声比约 10:1

**下一步**: 在完整数据集 (360题) 上验证
