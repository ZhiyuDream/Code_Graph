# 交接文档与当前实现对照

本文档将「项目交接说明（Code Repository Understanding）」与 **Code_Graph 仓库当前实现** 对齐，便于下一阶段接手或 AI 续写时知道：哪些已做、哪些未做、下一步建议做什么。

---

## 一、交接文档 vs 当前实现

| 交接文档描述 | 当前实现情况 |
|--------------|--------------|
| **函数级 Code Graph**：Function、File、Directory；Call、Containment、Include | **已实现**，但有差异：采用 **DESIGN 方案 A**，不建 Repository/File/Directory 节点，仅 **Function、Class、Variable**（带 file_path）；边为 **CALLS**、**CONTAINS**（Class→Function）、**REFERENCES_VAR**（Function→Variable）。目录/文件由 `file_path` 推导。 |
| AST parsing → call graph → 存储 | **已实现**：`run_stage1.py`（libclang）、`run_stage1_clangd.py`（clangd LSP）→ `graph_builder.py` → `neo4j_writer.py`。 |
| 函数调用关系检索、函数级语义搜索 | **已实现**：QA 流水线 `run_qa.py` 按意图路由 **A（Cypher 直接查）/ B（流程 BFS 展开）/ C（embedding + 邻域）**；Neo4j 查 CALLS、Function。 |
| **Benchmark 问题**：结构 / 流程 / 架构 / 设计 | **已对接**：`llama_cpp_QA.csv`（一级分类、问题类型、意图、实体、Evidence）；`docs/llama_cpp_QA_题目分类与检索策略.md` 将题目分为 A/B/C 与检索策略对应。 |
| **流程类问题**：需 workflow/调用链 | **部分实现**：类型 B 用「入度 0 的 Function 作流程起点 + LLM 选起点 + 沿 CALLS BFS 展开」近似 workflow，无显式 **Workflow 节点** 的自动发现与命名；阶段 2 有 `run_stage2.py` 写 Workflow 节点与 PART_OF_WORKFLOW，但 QA 当前未强制依赖该阶段。 |
| **模块级问题**：module abstraction | **未实现**：图中无 Module 节点；模块信息仅能通过 `file_path` 前缀或目录名推导。 |
| **方向一：Graph Node Annotation**（LLM 为节点生成 summary/role） | **未实现**：Function 节点无 summary/role；embedding 检索用「name \| file_path」拼接，未用 LLM 注释。 |
| **方向二：High-Level Abstraction**（Module、Workflow 自动发现） | **Workflow**：有阶段 2 入口展开与 Workflow 节点写入，未与 QA 深度打通；**Module**：未做。 |
| **流程入口定位**（入度低节点） | **已用**：类型 B 的 `_get_flow_start_candidates` 取「CALLS 入度 0、出度≥1」的 Function，再由 LLM 选 1～5 个起点。 |
| **Graph 质量评估**（retrieval recall、QA accuracy 等） | **已实现**：`eval_graph_coverage.py`（图对 Evidence 覆盖率、实体命中率）、`investigate_graph_scope.py`（Evidence/compile_commands/图路径对比）、`qa_stats.py`（正确率、证据召回/精确率、按问题类型/一级分类/路由类型统计）。 |

## 二、当前仓库关键路径（便于接手者快速定位）

- **建图**：`run_stage1.py` 或 `run_stage1_clangd.py` → `graph_builder.py` → `neo4j_writer.py`
- **阶段 2 Workflow**：`run_stage2.py`、`entry_candidates.py`
- **QA 流水线**：`run_qa.py`（路由 A/B/C、检索、LLM 生成、`--eval` 打分）、`qa_retrieval_results.json`
- **统计与图质量**：`qa_stats.py` → `qa_retrieval_stats.md`；`eval_graph_coverage.py` → `graph_coverage_report.md`；`investigate_graph_scope.py` → `graph_scope_investigation.md`
- **配置**：`config.py` 读 `.env`（Neo4j、REPO_ROOT、OPENAI_*、GITHUB_* 等）
- **设计与优化方向**：`DESIGN.md`、`docs/优化方向与待做项.md`、`docs/llama_cpp_QA_题目分类与检索策略.md`
