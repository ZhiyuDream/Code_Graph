# 目录驱动的 ReAct 检索方案（废弃 A/B/C）

## 核心思想

模仿人类专家阅读代码的过程：

```
1. 看整体结构（顶层目录）
2. 定位相关模块（根据问题关键词）
3. 深入目标目录（查看文件和函数）
4. 分析关键代码（调用链、依赖关系）
5. 如果不够，返回步骤 2 或 3 继续探索
```

这不是分类问题，而是**探索问题**。

---

## 工具层级（从粗到细）

```
层1: 仓库概览
  ├── 顶层目录结构
  └── 各目录功能描述（LLM 生成或预存）

层2: 目录探索  
  ├── 目录下的子目录/文件列表
  ├── 目录内核心函数（按 fan_in 排序）
  └── 目录功能摘要（LLM 实时生成）

层3: 文件分析
  ├── 文件内的函数/类列表
  ├── 函数源码（按需读取）
  └── 函数调用关系

层4: 调用链展开
  ├── 从函数出发的调用链
  └── 跨文件依赖分析
```

## ReAct 流程（循环探索）

```python
class DirectoryDrivenAgent:
    def __init__(self, driver):
        self.driver = driver
        self.context = {}  # 已收集的信息
        self.visited_dirs = set()  # 已访问目录
        
    def run(self, question: str) -> str:
        """主循环"""
        # Step 0: 提取问题关键词
        keywords = self.extract_keywords(question)
        
        # Step 1: 获取仓库概览
        repo_overview = self.get_repo_overview()
        self.context["overview"] = repo_overview
        
        # Step 2: 根据关键词定位候选目录
        candidate_dirs = self.rank_directories(keywords, repo_overview)
        
        # Step 3-6: 迭代探索（ReAct 循环）
        for step in range(MAX_STEPS):
            # Thought: 分析当前已收集的信息，决定下一步
            thought = self.llm_think(question, self.context)
            
            # Action: 选择工具执行
            action = self.decide_action(thought, candidate_dirs)
            
            if action["type"] == "explore_dir":
                # 深入探索某个目录
                result = self.explore_directory(action["dir_path"])
            elif action["type"] == "analyze_function":
                # 分析具体函数
                result = self.analyze_function(action["func_id"])
            elif action["type"] == "expand_calls":
                # 展开调用链
                result = self.expand_call_chain(action["func_id"])
            elif action["type"] == "answer":
                # 信息足够，生成答案
                break
            
            # Observation: 记录结果到 context
            self.context[f"step_{step}"] = {
                "thought": thought,
                "action": action,
                "result": result
            }
        
        # 生成最终答案
        return self.generate_answer(question, self.context)
```

## 关键工具实现

### 1. 获取仓库概览

```python
def get_repo_overview(self) -> dict:
    """获取仓库顶层结构"""
    # 顶层目录
    dirs = self.query("""
        MATCH (d:Directory)
        WHERE d.path NOT CONTAINS '/'
        RETURN d.path AS path, d.name AS name
        ORDER BY d.path
    """)
    
    # 每个顶层目录的核心函数
    for d in dirs:
        core_funcs = self.query("""
            MATCH (d:Directory {path: $path})-[:CONTAINS*]->(f:File)-[:CONTAINS]->(fn:Function)
            RETURN fn.name AS name, fn.fan_in AS fan_in
            ORDER BY fn.fan_in DESC
            LIMIT 5
        """, {"path": d["path"]})
        d["core_functions"] = core_funcs
    
    # LLM 生成目录功能描述
    overview_text = format_dirs_for_llm(dirs)
    descriptions = self.llm_describe_directories(overview_text)
    
    return {
        "top_dirs": dirs,
        "descriptions": descriptions
    }
```

### 2. 目录排序（匹配关键词）

```python
def rank_directories(self, keywords: list, overview: dict) -> list:
    """根据关键词对目录排序"""
    scored_dirs = []
    
    for d in overview["top_dirs"]:
        score = 0
        
        # 名称匹配
        for kw in keywords:
            if kw.lower() in d["name"].lower():
                score += 10
            if kw.lower() in d["path"].lower():
                score += 5
        
        # 核心函数匹配
        for func in d.get("core_functions", []):
            for kw in keywords:
                if kw.lower() in func["name"].lower():
                    score += 3
        
        # LLM 描述匹配
        desc = overview["descriptions"].get(d["path"], "")
        for kw in keywords:
            if kw.lower() in desc.lower():
                score += 2
        
        scored_dirs.append((d, score))
    
    # 按分数排序
    scored_dirs.sort(key=lambda x: x[1], reverse=True)
    return [d for d, _ in scored_dirs]
```

### 3. 深入探索目录

```python
def explore_directory(self, dir_path: str, depth: int = 1) -> dict:
    """深入探索某个目录"""
    self.visited_dirs.add(dir_path)
    
    # 子目录和文件
    contents = self.query("""
        MATCH (d:Directory {path: $path})-[:CONTAINS]->(child)
        RETURN labels(child)[0] AS type, 
               child.path AS path, 
               child.name AS name
        ORDER BY type, path
    """, {"path": dir_path})
    
    subdirs = [c for c in contents if c["type"] == "Directory"]
    files = [c for c in contents if c["type"] == "File"]
    
    # 目录内核心函数
    core_funcs = self.query("""
        MATCH (d:Directory {path: $path})-[:CONTAINS*]->(f:File)-[:CONTAINS]->(fn:Function)
        RETURN fn.name AS name, 
               fn.file_path AS file,
               fn.fan_in AS fan_in,
               fn.fan_out AS fan_out
        ORDER BY fn.fan_in DESC
        LIMIT 10
    """, {"path": dir_path})
    
    # LLM 生成目录功能摘要
    summary = self.llm_summarize_directory(dir_path, files, core_funcs)
    
    return {
        "path": dir_path,
        "subdirs": subdirs,
        "files": files,
        "core_functions": core_funcs,
        "summary": summary,
        "related_dirs": self.find_related_dirs(dir_path)  # 调用关系指向的其他目录
    }
```

## Thought-Action-Observation 示例

**问题**: "ggml-metal 后端的初始化流程是什么？"

**Step 1 - Thought**: 
用户问的是 "ggml-metal" 后端的初始化流程。我需要先找到相关目录。

**Step 1 - Action**: 
- 调用 `get_repo_overview()` 获取顶层目录

**Step 1 - Observation**: 
- 顶层目录: src/, ggml/, examples/, common/
- ggml/ 目录下有 metal/ 子目录，包含 ggml-metal.m, ggml-metal.h

**Step 2 - Thought**: 
找到了 ggml/metal/ 目录，需要深入查看其中的初始化相关函数。

**Step 2 - Action**: 
- 调用 `explore_directory("ggml/src")`

**Step 2 - Observation**: 
- 文件: ggml-metal.m, ggml-metal.h
- 核心函数: ggml_metal_init (fan_in=12), ggml_metal_graph_compute (fan_in=8)
- 摘要: "该目录包含 Metal GPU 后端实现，提供初始化和计算调度功能"

**Step 3 - Thought**: 
找到了 `ggml_metal_init` 函数，需要查看它的调用链来了解完整流程。

**Step 3 - Action**: 
- 调用 `analyze_function("ggml_metal_init")`
- 调用 `expand_call_chain("ggml_metal_init", depth=3)`

**Step 4 - Thought**: 
已收集到足够信息：入口函数、调用链、相关文件。

**Step 4 - Action**: 
- 调用 `generate_answer()`

**Answer**: 
"ggml-metal 后端的初始化流程如下：
1. 入口: `ggml_metal_init()` (ggml/src/ggml-metal.m:123)
2. 步骤: 创建设备 → 申请缓冲区 → 加载 shader
3. 调用关系: init → new_buffer → load_shader → ..."
