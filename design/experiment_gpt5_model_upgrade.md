# 实验: GPT-5.4 模型升级测试

## 日期
2026-04-22

## 改进内容
将 LLM 模型从 `gpt-4o-mini` 升级为 `gpt-5.4`：
- `.env` 中 `LLM_MODEL=gpt-5.4`
- 修复 API 兼容性问题：`max_tokens` → `max_completion_tokens`
- 修复文件: `tools/core/llm_client.py`, `tools/eval_benchmark.py`

## 实验配置
- 数据集: 360题
- 检索配置: V8 + File Expansion + Grep V2（当前最佳配置）
- 对照组: 同配置 + gpt-4o-mini（74.4% / 74.7%）

## 结果

### 准确率对比
| 配置 | 模型 | 准确率 | 平均分数 |
|---|---|---|---|
| V8 + FileExp + GrepV2 | gpt-4o-mini | **74.4%** | 0.682 |
| V8 + FileExp + GrepV2 | **gpt-5.4** | **23.9%** | **0.461** |

### 分数分布对比
| 分数段 | gpt-4o-mini | gpt-5.4 |
|---|---|---|
| [0.0, 0.3) 差 | 1.9% | **19.7%** |
| [0.3, 0.5) 较差 | 3.3% | **31.7%** |
| [0.5, 0.7) 一般 | 32.8% | 24.2% |
| [0.7, 0.9) 良好 | **50.6%** | 21.4% |
| [0.9, 1.0) 优秀 | **11.4%** | 3.1% |

## 关键发现

### 1. GPT-5.4 不是变差了，而是回答风格根本不同

**gpt-4o-mini 风格**（被判正确时）：
> "你问的是 xxx，基于检索信息，我做如下详细分析..."

**GPT-5.4 风格**（被判错误时）：
> "从你给的信息看，这个问题的核心不是...而是...当前检索结果里，和 xxx 直接强相关的只有..."

GPT-5.4 更谨慎、更爱做元分析，但旧 judge 把这种谨慎解读为"超出范围"和"臆测"。

### 2. GPT-5.4 作为 judge 更严格

用 GPT-5.4 重新评判 gpt-4o-mini 的答案（前5题）：
- 旧 judge: 4✅ 1❌
- GPT-5.4 judge: **0✅ 5❌**

GPT-5.4 发现了旧模型答案中同样存在的"推断过度"问题，只是旧 judge 没有识别出来。

### 3. 根本问题：Prompt 没有约束模型

当前 `generate_answer` 的 prompt 没有明确要求：
- "只基于检索到的信息回答"
- "不要推断超出检索范围的内容"
- "直接回答问题，不要先分析问题的意图"

这导致两个模型都在推断，但 GPT-5.4 的推断被更严厉地惩罚。

## 结论

**GPT-5.4 升级暂时失败，准确率从 74.4% 暴跌至 23.9%**

原因不是模型能力差，而是：
1. GPT-5.4 回答风格更"分析性"、更发散
2. 旧 judge 对这种风格惩罚严重
3. Prompt 没有有效约束模型只基于证据回答

## 下一步建议

### 方案A：优化 Answer Generator Prompt（推荐）
在 `generate_answer` 中加入严格约束：
```
- 只基于【已检索到的函数和Issue】回答问题
- 不要推断检索结果之外的信息
- 不要先分析"问题的核心是什么"，直接给出答案
- 如果信息不足，直接说明"根据现有信息无法确定"
```

### 方案B：用 GPT-5.4 作为 Judge 重新评估所有历史结果
如果 GPT-5.4 作为 judge 更严格、更准确，应该用统一标准重新评估，避免旧 judge 的宽松偏差。

### 方案C：回退模型（不推荐）
回退到 gpt-4o-mini，但放弃了 GPT-5.4 的推理能力优势。
